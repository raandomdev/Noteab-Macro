# 2.1.7 (Whats new?)

## Note: This macOS porting is expect to be with bugs, pleace report any bugs to @spacedev0572 and report your problem. note that not all features its expected to work, since there are a lot of limitations to work with. Please also note that this macOS porting will be released at its own time. This means that every Coteab windows update will not be released at the same time. Thanks for your understanding!

## Features:
* macOS Porting by spacedev0572!
* Eden pathing + auto contract **(EXPERIMENTAL - EDEN PATH MIGHT FAIL IF YOU GOT LAGSPIKE A LOT)** **(NEW)**
* Only allow one Coteab instance running (opening another Coteab will show you a warning popup)
* Attempt fix at autopop bug **(hopefully... this is the 3rd time I'm fixing this :sob:)**
* New remote access control commands **(start, stop, status, equip aura, close_roblox_and_stop_macro)**
* Individual Discord ping (userid/roleid) for each biome **(Non-rare biomes CANNOT be assigned @everyone/@here obviously...)**
* "Status" page added -> Allows you to see which macro actions are currently enabled/idle/off
* Alternative fallback mode for anyone who got pure black/white screen -> now they can open the macro in browser using **Ctrl + Shift + F10**

## Changes:
* Fixed biome config data desync
* Macro_logs.txt refactored (it now logs almost everything in the macro's internal logic)
* Configs & files relocated to %appdata% (you can open the appdata folder in the "Other Features" section)
